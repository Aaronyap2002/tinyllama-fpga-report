% extreport is the report class from the extsizes package.
% It supports a fixed set of sizes: 8pt 9pt 10pt 11pt 12pt 14pt 17pt 20pt.
% We use 11pt as the closest available size to the NTU template's 10.5pt.
% Unlike scrreprt (KOMA-Script), extreport behaves exactly like the standard
% report class -- all standard packages work with it without conflicts.
\documentclass[12pt, a4paper]{extreport}

% -- Core font (Times New Roman) ----------------------------------------------
% mathptmx switches both the text font and the math font to Times New Roman.
\usepackage{mathptmx}

% -- Page geometry ------------------------------------------------------------
% Left margin is wider (3.5cm) to match the NTU Word template binding margin.

  a4paper,
  left       = 3.5cm,
  right      = 3.0cm,
  top        = 3.0cm,
  bottom     = 3.0cm,
  headheight = 14pt,
  headsep = 12pt
]{geometry}

% -- Line spacing -------------------------------------------------------------
% The NTU Word template uses 1.5x line spacing throughout the body text.
% With extreport (unlike scrreprt), calling \onehalfspacing directly in the
% preamble is perfectly safe -- there is no internal geometry recalculation
% that conflicts with setspace's font metric changes.
\usepackage{setspace}

% -- Headers and footers (fancyhdr) -------------------------------------------
% fancyhdr works cleanly with extreport because extreport is a standard class.
% The conflict warnings we saw before were specific to using fancyhdr with
% KOMA-Script's scrreprt, which has its own incompatible header/footer system.
\usepackage{fancyhdr}
\pagestyle{fancy}
\fancyhf{}                                   % clear all header/footer fields
\fancyhead[L]{\small\leftmark}               % chapter name on left of header
\fancyhead[R]{\small Project No: A2285-251} % project number on right of header
\fancyfoot[C]{\thepage}                      % page number centred in footer
\renewcommand{\headrulewidth}{0pt}           % no horizontal rule under header
\renewcommand{\footrulewidth}{0pt}           % no horizontal rule above footer

% Redefine the 'plain' style used on chapter-opening pages.
% By default, 'plain' shows only a centred page number and nothing else.
% We override it here so chapter-opening pages also show the project number,
% keeping the header consistent throughout the document.
\fancypagestyle{plain}{
  \fancyhf{}
  \fancyhead[R]{\small Project No: A2285-251}
  \fancyfoot[C]{\thepage}
  \renewcommand{\headrulewidth}{0pt}
  \renewcommand{\footrulewidth}{0pt}
}

% A minimal page style exclusively for the cover page.
% It shows the project number in the top-right header (matching every other
% page) but deliberately defines no footer, so no page number is printed.
% The titlepage environment sets page style to 'empty' by default -- we
% override that with \thispagestyle{coverpage} inside Title/titlepage.tex.
\fancypagestyle{coverpage}{
  \fancyhf{}                                   % start from a clean slate
  \fancyhead[R]{\small Project No: A2285-251} % project number only
  \renewcommand{\headrulewidth}{0pt}           % no rule
  \renewcommand{\footrulewidth}{0pt}           % no rule
  % No \fancyfoot line -- this is intentional. The absence means no page
  % number is printed on the cover page.
}

% -- Chapter and section heading style (titlesec) -----------------------------

During the completion of this Final Year Project, I gained substantial knowledge in reconfigurable computing and hardware accelerator design for deep learning inference. I learned to apply High-Level Synthesis concepts such as loop pipelining, array partitioning, and AXI interface management to translate C++ algorithms into synthesisable hardware. Learning to use Vitis HLS 2021.2 and the Xilinx ZCU102 platform is an important outcome, as both tools are representative of the FPGA accelerator design flow used in the embedded systems and data-centre industry.
